# Claude Code × テスト設計 トレーニング 開発環境構築マニュアル

本マニュアルは、AI（Claude Code）を活用したテスト設計を行うための PC セットアップ手順書です。STEP 1 から順番に進めてください。

---

## STEP 1：VS Code（Visual Studio Code）のインストールと初期設定

プログラムを書いたり、各種ツールを操作したりするためのメインのコードエディタです。最初にインストールし、日本語環境を整えます。

### 1. インストール手順

1. [VS Code 公式サイト](https://code.visualstudio.com/) にアクセスし、「Download for Windows」をクリックします。
   管理者権限なしでインストールできる「User Installer (x64)」が自動的にダウンロードされます。
2. ダウンロードしたファイルを実行し、「次へ」を進めます。
3. 「追加タスクの選択」画面が出たら、以下の **すべてにチェック** を入れてください。
   - デスクトップ上にアイコンを作成する
   - エクスプローラーのファイル コンテキスト メニューに「Code で開く」追加
   - エクスプローラーのディレクトリ コンテキスト メニューに「Code で開く」追加
   - サポートされているファイルの種類のエディターとして登録する
   - PATH への追加
4. インストールを完了させ、VS Code を起動します。

> **補足**
> - 「Code で開く」の追加：フォルダを右クリックして VS Code を直接開けるようになります。
> - 「PATH への追加」：ターミナルから `code` コマンドで VS Code を起動できるようになる必須設定です。

### 2. 日本語化とセキュリティ設定

1. VS Code 左側の拡張機能アイコン（四角が 4 つ並んだアイコン）をクリックします。
2. 検索窓に `Japanese` と入力し、Microsoft 公式の「Japanese Language Pack for Visual Studio Code」を「Install」します。
3. 右下に「Change Language and Restart」と表示されたらクリックして再起動すると、日本語化が完了します。
4. セキュリティ対策として、設定画面（`Ctrl + ,`）を開き、検索窓に `Auto Update` と入力します。「All Extensions」に設定されているか確認してください（古い拡張機能の脆弱性リスクを防ぐためです）。

## STEP 2：Node.js のインストール

JavaScript を PC 上で直接実行するための実行環境です。Playwright など、本研修で使うツールの動作に必要です。

1. [Node.js 公式サイト](https://nodejs.org/ja/download/) にアクセスします。
2. 「LTS（推奨版）」の Windows 用インストーラーをダウンロードし、実行してすべて「Next（次へ）」でインストールを完了させます。

**確認**

VS Code のターミナル（上部メニュー「ターミナル」→「新しいターミナル」）を開き、以下のコマンドを実行してください。

```
node -v
```

バージョン番号（例：`v22.x.x`）が表示されれば成功です。

## STEP 3：Git のインストールと GitHub への接続設定

ソースコードの変更履歴を管理し、GitHub でチームと共有するための必須ツールです。

### 1. インストール手順

1. [Git 公式サイト](https://git-scm.com/) から「Download for Windows」→「64-bit Git for Windows Setup」をダウンロードして実行します。
2. 設定項目は多数ありますが、以下の項目のみ変更し、それ以外は「Next」で進めてください。

| 設定項目 | 選択内容 |
|---|---|
| 初期ブランチ名 | 「Override the default branch name for new repositories」を選び、`main` と入力 |
| PATH の設定 | 「Git from the command line and also from 3rd-party software」を選択 |
| SSH クライアント | 「Use bundled OpenSSH」を選択 |
| HTTPS 通信の SSL | 「Use the native Windows Secure Channel library」を選択 |
| 改行コード | 「Checkout Windows-style, commit Unix-style line endings」を選択 |
| ターミナル | 「Use MinTTY」を選択 |
| git pull の動作 | 「Fast-forward or merge」を選択 |
| 認証情報 | 「Git Credential Manager」を選択 |
| 追加オプション | 「Enable file system caching」のみチェック |

> **補足**
> - ブランチ名を `main` にする：現在の GitHub の標準ルールに合わせるためです。
> - 改行コードの変換：Windows では CRLF で編集し、コミット時に自動で LF（Unix/Linux 標準）に変換することで、Docker 環境等でのエラーを防ぎます。
> - Git Credential Manager：GitHub との認証トークンを安全に保存し、毎回認証する手間を省きます。

### 2. インストール確認と初期設定

1. 開いている VS Code やターミナルをすべて閉じ、VS Code を再起動してください（PATH の設定を反映させるために必要です）。
2. VS Code を開き、上部メニューの「ターミナル」→「新しいターミナル」を開きます。
3. 以下のコマンドを実行し、バージョンが表示されれば成功です。

```
git --version
```

4. 続けて以下のコマンドを 1 行ずつ実行し、ユーザー情報を登録します。`名前` の部分は半角英語（例：`Taro Yamada`）、`メールアドレス` の部分はご自身のものに書き換えてから実行してください。

```
git config --global user.name "名前"
git config --global user.email "メールアドレス"
```

### 3. GitHub アカウントの準備

GitHub はソースコードをインターネット上で管理・共有するためのサービスです。

1. [GitHub 公式サイト](https://github.com/) にアクセスし、アカウントをお持ちでない場合は「Sign up」から無料アカウントを作成してください。
2. GitHub への認証設定は、次のセクション「4. GitHub CLI のインストールと認証」で GitHub CLI を使ってまとめて行います。

### 4. GitHub CLI のインストールと認証

GitHub CLI（`gh` コマンド）は、ターミナルからプルリクエスト（PR）の作成やリポジトリ操作を行うためのツールです。本研修の演習課題で使用します。

`gh auth login` で認証を完了させると、`git push` や `git clone` などの git コマンドによる GitHub.com への通信も含め、認証が GitHub CLI に一本化されます。

**インストール**

以下のいずれかの方法でインストールします。

**方法 A：winget を使う（推奨）**

VS Code のターミナルで以下を実行します。

```
winget install --id GitHub.cli
```

**方法 B：インストーラーを使う**

1. [GitHub CLI リリースページ](https://github.com/cli/cli/releases/latest) から `gh_X.X.X_windows_amd64.msi`（X.X.X は最新バージョン番号）をダウンロードして実行します。
2. インストーラーの指示に従い「Finish」まで進めます。

**インストール確認**

インストール後、**VS Code を再起動**してからターミナルで以下を実行します。

```
gh --version
```

バージョン番号（例：`gh version 2.x.x`）が表示されれば成功です。

**GitHub への認証**

以下のコマンドを実行し、対話形式で認証を進めます。

```
gh auth login
```

質問が表示されたら、以下の順で選択します。

| 質問 | 選択内容 |
|---|---|
| Where do you use GitHub? | `GitHub.com` |
| What is your preferred protocol? | `HTTPS` |
| Authenticate Git with your GitHub credentials? | `Y` |
| How would you like to authenticate? | `Login with a web browser` |

ブラウザが開いてデバイスコードの入力画面が表示されます。ターミナルに表示されたコードを入力し、「Authorize github-cli」をクリックして認証を完了させます。

ターミナルに戻り、以下のコマンドで認証状態を確認します。

```
gh auth status
```

以下のように `✓ Logged in to github.com account <アカウント名>` と表示されれば設定完了です（`<アカウント名>` の部分にはご自身の GitHub アカウント名が、末尾の括弧内は環境によって異なる文字列が表示されます）。

```
github.com
  ✓ Logged in to github.com account <アカウント名> (keyring)
```

> **トラブルシューティング**
> 認証をリセットしたい場合は、VS Code のターミナルで以下を実行します。
> ```
> gh auth logout
> ```
> その後、再度 `gh auth login` を実行して再認証できます。

## STEP 4：VS Code 拡張機能のインストール

VS Code 左側の拡張機能アイコンから、以下のツールを検索してインストールします。

### 1. Claude Code（AI エージェント）

テキストで指示するだけでテストケース生成・コード生成・コードレビューを行う AI ツールです。本研修の中心となります。最初にインストールし、ログインまで完了させてください。

1. 検索窓に `Claude Code` と入力し、作者が **Anthropic** のものをインストールします。
2. `Ctrl + Shift + P` でコマンドパレットを開き、`Claude Code` と入力して「Open in New Tab」を選択すると起動します。
3. 初回は Anthropic アカウントでのログインが必要です。

### 2. Mermaid（図解作成ツール）

Mermaid は、テキストでフローチャート・状態遷移図・シーケンス図などを記述・生成できるダイアグラムツールです。テキストファイルとして保存されるため、Git でのバージョン管理が可能です。以下の 3 つをインストールします。

**① Markdown Preview Mermaid Support**
*役割：Markdown のプレビュー画面に Mermaid の図を描画する*

1. 検索窓に `Markdown Preview Mermaid Support` と入力し、作者が **Matt Bierner** のものをインストールします。
2. Mermaid コードブロックを含む Markdown ファイルを開き、`Ctrl + Shift + V` でプレビューを開くと図が表示されます。

**② Mermaid Markdown Syntax Highlighting**
*役割：Mermaid コードに色をつけて読みやすくする*

1. 検索窓に `Mermaid Markdown Syntax Highlighting` と入力し、作者が **Brian Pruitt-Goddard** のものをインストールします。
2. インストール後、Markdown ファイル内の Mermaid コードブロックに自動的にシンタックスハイライトが適用されます。追加の操作は不要です。

**③ Mermaid Graphical Editor**
*役割：コードを書かずにマウス操作で図を編集できる GUI を提供する*

1. 検索窓に `Mermaid Graphical Editor` と入力し、作者が **corschenzi** のものをインストールします。
2. `Ctrl + Shift + P` でコマンドパレットを開き、`Mermaid` と入力すると図の種類ごとの GUI エディタを起動できます。

### 3. Postman（API テストツール）

Web サービスの API（サーバーとやり取りするための接続口）にリクエストを送信し、正しいレスポンスが返るかを確認するテストツールです。VS Code 上で使用できます。

1. 検索窓に `Postman` と入力し、作者が **Postman** のものをインストールします。
2. インストール後、左側のサイドバーに追加された Postman アイコンをクリックし、お持ちのアカウント（Google アカウント等）でサインインしてください。

> **トラブルシューティング**
> Postman の画面が真っ白になる場合は、設定画面（`Ctrl + ,`）から `Http: Proxy Support` を検索し、`override` に変更すると解消されます。

### 4. Playwright（E2E テスト自動化）

ブラウザを自動操作し、ボタンのクリック・フォーム入力・画面の確認などを繰り返し実行する E2E（End-to-End）テスト自動化ツールの VS Code 拡張機能です。

1. 検索窓に `Playwright Test for VSCode` と入力し、作者が **Microsoft** のものをインストールします。
2. インストール後、VS Code 左側のサイドバーにビーカー型のアイコン（テストエクスプローラー）が追加されます。クリックするとテスト一覧が表示されます。
3. `Ctrl + Shift + P` でコマンドパレットを開き、`Testing: Open Test Explorer` と入力しても起動できます。

## STEP 5：トレーニング用ワークスペースを開き、Playwright 本体をインストールする

STEP 4 でインストールした拡張機能は VS Code 上の操作画面（UI）を提供するものです。テストを実際に実行するには、プロジェクトに Playwright 本体（npm パッケージ）を別途インストールする必要があります。先にトレーニング用のワークスペースを開いてから実行してください。

### 1. ワークスペースを開く

1. VS Code のメニューから「ファイル」→「ファイルでワークスペースを開く」を選択します。
2. 以下のファイルを選択して開きます。

```
C:\dev\Visual Studio Code\claude-testing-training\claude-testing-training.code-workspace
```

### 2. Playwright 本体のインストール

VS Code のターミナルを開き（上部メニュー「ターミナル」→「新しいターミナル」）、以下の 2 つのコマンドを順番に実行してください。

**① npm パッケージのインストール**

```
npm install
```

`added XX packages` のような表示が出れば成功です。

> **Playwright 課題のみ取り組む受講者へ（リポジトリをクローンしていない場合）**
>
> `npm install` を実行したときに `npm error` や `ENOENT`（`package.json` が見つからない）と表示された場合は、`package.json` が存在しません。以下の手順で先に作成してください。
>
> 1. VS Code のエクスプローラー（左サイドバー）でプロジェクトのルートを右クリックし、「新しいファイル」を選択します。
> 2. ファイル名を `package.json` とします。
> 3. 以下の内容をそのまま貼り付けて保存してください。
>
> ```json
> {
>   "name": "claude-testing-training",
>   "version": "1.0.0",
>   "scripts": {
>     "test": "playwright test",
>     "test:headed": "playwright test --headed",
>     "test:report": "playwright show-report"
>   },
>   "devDependencies": {
>     "@playwright/test": "^1.44.0",
>     "typescript": "^5.4.0"
>   }
> }
> ```
>
> 4. 改めて `npm install` を実行してください。

**② ブラウザ本体のインストール**

```
npx playwright install chromium
```

Chromium ブラウザがダウンロードされます。完了まで数分かかる場合があります。

## STEP 6：WSL2（ローカル Ubuntu）のセットアップ

WSL2（Windows Subsystem for Linux 2）は、Windows 上で Linux 環境を動かす機能です。Linux コマンドの練習では、この機能で PC 内に作った Ubuntu 環境を使います。クラウドサービスを使わず、自分の PC 上だけで完結します。

### 1. WSL2 の有効化と Ubuntu のインストール

1. スタートボタンを右クリックし、「ターミナル（管理者）」を起動します。
2. 以下のコマンドを実行します。

```
wsl --install
```

3. コマンドの実行が終わったら、**PC を再起動**します。

4. 再起動後、スタートボタンを右クリック →「ターミナル（管理者）」を開き、以下のコマンドで Ubuntu のインストール状況を確認します。

```
wsl --list --verbose
```

結果に応じて次の操作が異なります。

**`Ubuntu` が一覧に表示された場合：** 以下のコマンドを実行して Ubuntu を起動します。

```
ubuntu
```

**「インストールされているディストリビューションはありません」と表示された場合：** 以下のコマンドを実行します。ターミナル上でインストールが自動的に進み、完了後に Ubuntu が起動します。

```
wsl --install -d Ubuntu
```

5. Ubuntu が起動すると、次の手順で初回設定が始まります。

**① ユーザー名の設定**

`Enter new UNIX username:` と表示されたら、Ubuntu で使うユーザー名（半角英小文字で始まる英数字）を入力して `Enter` を押します。

**② パスワードの設定**

`New password:` と表示されたらパスワードを入力して `Enter` を押します。続けて `Retype new password:` が表示されるので、同じパスワードをもう一度入力します。入力中は画面に表示されませんが、正しく入力されています。

**③ 使用状況データ送信の設定**

`Would you like to opt-in to platform metrics collection (Y/n)?` と表示されたら、`n` を入力して `Enter` を押します。

`（ユーザー名）@（PC 名）:~$` のような表示に切り替われば、Ubuntu のセットアップ完了です。

### 2. VS Code から Ubuntu 内のファイルを操作する（参考）

> **本研修の演習課題ではこの手順を使用しません。**
> 演習では VS Code のターミナルで `wsl` コマンドを実行して Ubuntu を操作します。VS Code を WSL に接続すると Claude Code が利用できなくなるため、本研修では使用しません。

Ubuntu 内のプロジェクトを VS Code で直接開いて編集したい場面では、WSL 拡張機能と WSL 接続を組み合わせて使います。

**① WSL 拡張機能のインストール**

1. VS Code 左側の拡張機能アイコンをクリックします。
2. 検索窓に `WSL` と入力し、作者が **Microsoft** のものをインストールします。

**② VS Code と Ubuntu の接続**

1. VS Code 左下のリモートアイコン（`><`）をクリックします。
2. **[WSL への接続]** を選択します。
3. VS Code 画面左下のステータスバーに **[WSL: Ubuntu]** と表示されれば接続完了です。

> **トラブルシューティング**
> **[WSL への接続]** が表示されない場合は、VS Code を一度閉じて再起動してください。拡張機能のインストール後に再起動が必要な場合があります。
