# 課題3：ATRSのAPIをPostmanでテストする

**所要時間：** 20〜30 分  
**対象：** 「APIテストとPostman入門」を読み終えた受講者  
**前提：**
- ATRSがローカルで起動していること（`http://localhost:8082/atrs/`）
- VS Code の Postman 拡張機能にサインイン済みであること

---

## この課題の目的

JSONPlaceholderで練習したPostmanの操作を、実際のATRSに対して使います。  
この課題のゴールは「ATRSのAPIにリクエストを送り、レスポンスを確認できること」です。  
テスト設計は行いません。まず動かしてみることに集中してください。

---

## ① 会員番号確認API（GETリクエスト）

ATRSには「指定した会員番号が存在するかどうか」を確認するAPIがあります。  
まずこのAPIで、ローカルのATRSに繋がることを確認しましょう。

### 手順1：存在する会員番号で確認する

```
HTTPメソッド：GET
エンドポイント：http://localhost:8082/atrs/api/member/available
クエリパラメータ：?membershipNumber=0000000001
```

**操作手順：**

1. HTTPメソッドのドロップダウンから **GET** を選択する
2. URLフィールドに `http://localhost:8082/atrs/api/member/available?membershipNumber=0000000001` を貼り付ける
3. **[Send]** をクリックする
4. レスポンスエリアの **Status** でステータスコードを確認する

**確認チェックリスト：**

- [ ] ステータスコードが **200** であることを確認した
- [ ] レスポンスボディが空であることを確認した

> **メモ：** このAPIはレスポンスボディを返しません。「会員が存在する」という情報は、ステータスコードだけで表現されています。

---

### 手順2：存在しない会員番号で確認する

```
HTTPメソッド：GET
エンドポイント：http://localhost:8082/atrs/api/member/available
クエリパラメータ：?membershipNumber=9999999999
```

**操作手順：**

1. HTTPメソッドのドロップダウンから **GET** を選択する
2. URLフィールドに `http://localhost:8082/atrs/api/member/available?membershipNumber=9999999999` を貼り付ける（`membershipNumber` の値が `9999999999` になっていることを確認する）
3. **[Send]** をクリックする
4. レスポンスエリアの **Status** でステータスコードを確認する

**確認チェックリスト：**

- [ ] ステータスコードが **404** であることを確認した

> **気づきポイント：** 手順1では200、手順2では404が返りました。  
> このAPIは「存在する → 200」「存在しない → 404」という設計になっています。  
> ステータスコードが情報を運ぶことを確認できました。

## ② 空席照会API（GETリクエスト＋クエリパラメータ）

次に、空席照会APIを試します。  
このAPIはJSONの配列をレスポンスとして返します。

### 手順3：東京(羽田)→大阪(伊丹)の空席を照会する

以下のエンドポイントにクエリパラメータを付けてリクエストします。`depDate` の `YYYY/MM/DD` の部分だけ、**今日から3日後の日付**に書き換えてから使用してください。

```
HTTPメソッド：GET
エンドポイント：http://localhost:8082/atrs/api/flights
```

**クエリパラメータ：**

| パラメータ | 値 | 意味 |
|---|---|---|
| `flightType` | `OW` | 片道（One Way） |
| `depAirportCd` | `HND` | 出発：東京(羽田) |
| `arrAirportCd` | `ITM` | 到着：大阪(伊丹) |
| `depDate` | `YYYY/MM/DD` | 搭乗日（スラッシュ区切り） |
| `boardingClassCd` | `N` | 一般席（Normal） |

**操作手順：**

1. 上の表の `depDate` を今日から3日後の日付に置き換え、以下のURLを作成する（例：`2026/05/14`）  
   `http://localhost:8082/atrs/api/flights?flightType=OW&depAirportCd=HND&arrAirportCd=ITM&depDate=2026/05/14&boardingClassCd=N`
2. HTTPメソッドのドロップダウンから **GET** を選択する
3. URLフィールドに作成したURLを貼り付ける
4. **[Send]** をクリックする
5. レスポンスエリアの **Pretty** タブでJSONを確認する

**確認チェックリスト：**

- [ ] ステータスコードが **200** であることを確認した
- [ ] レスポンスがJSONの **配列**（`[...]`）であることを確認した
- [ ] 各要素に `flightName`（便名）と `depTime`（出発時刻）が含まれていることを確認した
- [ ] `fareTypes` というキーがあることを確認した

> **補足：** URLを貼り付けると、Postmanが自動でクエリパラメータを解析し、**Params** タブに各パラメータを一覧表示します。入力内容の確認に使えます。

> **気づきポイント：** `fareTypes` の中には複数の運賃種別（片道運賃、予約割1 など）が入っています。  
> このデータは「課題1」で学んだ運賃算出のルールが反映されたものです。
