# 課題7：Playwrightで自動テストを動かしてみる

**所要時間：** 25〜35 分  
**対象：** 課題6を終えた受講者  
**前提：** [Playwright入門.md](../学習資料/Playwright入門.md) を読んでいること

---

## この課題の目的

この課題では、コードは一切書きません。

Claude Code にテストコードを生成させて、そのまま実行します。  
**自動テストが動く様子を目で見て、結果を成果物として記録する**のがゴールです。

---

## 前提：ATRSを起動しておく

テスト実行前に、ローカルのATRS（正常版）が起動していることを確認してください。  
`http://localhost:8082/atrs/auth/login?form` にブラウザでアクセスできれば準備OKです。

---

## Step 1：Claude Code にテストコードを生成させる（5〜7分）

Claude Code を起動し、以下のプロンプトをそのままコピーして送ってください。

```
以下の条件で playwright.config.ts と tests/login.spec.ts を作成してください。

【playwright.config.ts】
- テストフレームワーク：Playwright（TypeScript）
- baseURL：http://localhost:8082/atrs/
- ブラウザ：Chromium のみ
- ヘッドレスモード：true（デフォルト）

【tests/login.spec.ts】
- ログイン画面のURLパス：auth/login?form
- 会員番号入力欄のid：membershipNumber
- パスワード入力欄のid：password
- ログインボタンのid：login-btn
- テスト用会員番号：0000000001 / パスワード：aaaaa11111

テストケースは docs/演習/学習資料/Playwright入門.md の
「Part 6：課題5との接続」の表（TC-L-001〜TC-L-005）を参照してください。
```

> **playwright.config.ts がすでにある場合**  
> 以前に Playwright を初期化したことがある場合などに、ファイルがすでに存在していることがあります。  
> Claude Code が上書きを確認してきたら「上書きしてください」と伝えてください。  
> ATRSに合わせた設定に置き換わるので問題ありません。

Claude Code がファイルを作成したら完了です。中身は読まなくて構いません。

---

## Step 2：ブラウザが動く様子を見る（5〜7分）

VS Code のターミナルを開き（`Ctrl+Shift+@`）、以下のコマンドを実行してください。

```
npm run test:headed
```

ブラウザが自動的に起動し、ログイン画面が次々と操作される様子が見えます。  
コマンドが終わるまで、そのまま眺めていてください。

---

## Step 3：結果を確認する（2〜3分）

ブラウザが閉じると、ターミナルに結果が表示されます。

```
Running 5 tests using 1 worker

  ✓  1 [chromium] › tests\login.spec.ts:22:7 › ログイン画面 (A101) - 状態遷移テスト › TC-L-001: 正常ログイン (1.3s)
  ✓  2 [chromium] › tests\login.spec.ts:30:7 › ログイン画面 (A101) - 状態遷移テスト › TC-L-002: クライアントエラー（会員番号 未入力） (0.8s)
  ✓  3 [chromium] › tests\login.spec.ts:38:7 › ログイン画面 (A101) - 状態遷移テスト › TC-L-003: クライアントエラー（パスワード 未入力） (0.6s)
  ✓  4 [chromium] › tests\login.spec.ts:46:7 › ログイン画面 (A101) - 状態遷移テスト › TC-L-004: 認証エラー（会員番号 不存在） (1.5s)
  ✓  5 [chromium] › tests\login.spec.ts:54:7 › ログイン画面 (A101) - 状態遷移テスト › TC-L-005: 認証エラー（パスワード 不一致） (2.1s)

  5 passed (7s)
```

`[chromium]` はテストを実行したブラウザ名です。  
各行に `✓` が表示され、最後に `5 passed` が出ていれば成功です。

> **表示される秒数は環境によって異なります。** 気にしなくて大丈夫です。

### HTML レポートを開く

続けて以下のコマンドを実行してください。ブラウザで詳細な HTML レポートが開きます。

```
npm run test:report
```

各テストケースの実行時間・成否が一覧で確認できます。  
テストケース名をクリックすると、各操作と確認の詳細も確認できます。

確認できたらブラウザを閉じ、ターミナルで `Ctrl+C` を押してサーバーを停止してから次のステップへ進んでください。

---

## Step 4：テスト結果をドキュメントにまとめる（3〜5分）

以下のプロンプトを Claude Code に送ってください。

```
テスト実行結果を docs/演習/結果/課題7 テスト実行結果.md として作成してください。
以下の内容を含めてください。
- 実行日
- テストケース一覧と結果（パス／フェイル）
- 合計実行時間
- 使用したブラウザ
```

Claude Code がファイルを作成したら内容を確認してください。

---

## Step 5：Claude Code に Git 操作を依頼する（3〜5分）

テストが通ったことを確認したら、成果物を Git で記録しましょう。  
以下のプロンプトを Claude Code に送ってください。

```
現在いるブランチを起点に feature/exercise-7 を作成し、以下の3ファイルをコミットしてください。
- playwright.config.ts
- tests/login.spec.ts
- docs/演習/結果/課題7 テスト実行結果.md

コミットメッセージは適切なものを考えてください。
origin へのプッシュと、GitHub で feature/exercise-7 から main へのプルリクエスト作成まで行ってください。
```

> **`gh` コマンドが使えない場合**  
> Claude Code が「`gh` コマンドが見つかりません」と表示した場合は、ブラウザで以下の URL を開いて PR を作成してください。  
> `https://github.com/（自分のアカウント名）/claude-testing-training/pull/new/feature/exercise-7`  
> base を `main`、compare を `feature/exercise-7` に設定して「Create pull request」をクリックすれば完了です。

```typescript
import { test, expect, Page } from '@playwright/test';

// ────────────────────────────────────────────────────────────────
// 共通処理：ログイン画面を開く
//
// playwright.config.ts に baseURL = 'http://localhost:8082/atrs/' が設定されているため、
// page.goto('auth/login?form') は http://localhost:8082/atrs/auth/login?form に遷移する。
//
// goto の直後に URL を確認しているのは「画面が正しく表示された状態」でテストを始めるため。
// この確認が失敗した場合は、ATRS は起動しているが期待と異なる URL にリダイレクトされている可能性がある。
// ATRS 自体が起動していない場合は、page.goto() の時点で接続エラーになりこの行には到達しない。
// ────────────────────────────────────────────────────────────────
async function gotoLoginPage(page: Page) {
  await page.goto('auth/login?form');
  await expect(page).toHaveURL(/\/auth\/login/);
}

// ────────────────────────────────────────────────────────────────
// 共通処理：ログインフォームに入力してログインボタンをクリックする
//
// 引数に空文字列を渡した場合は fill をスキップする。
// これにより「会員番号 未入力」「パスワード 未入力」のテストを同じ関数で表現できる。
//
// セレクタ（#membershipNumber, #password, #login-btn）は
// ATRS のログイン画面 HTML の id 属性に対応している。
// ────────────────────────────────────────────────────────────────
async function submitLoginForm(page: Page, membershipNumber: string, password: string) {
  if (membershipNumber) await page.fill('#membershipNumber', membershipNumber);
  if (password)         await page.fill('#password', password);
  await page.click('#login-btn');
}

// ════════════════════════════════════════════════════════════════
// ログイン画面 (A101) 状態遷移テスト
//
// テストベース：課題5で作成した状態遷移図（カバレッジアイテム T1〜T3）
// 各テストケース（TC-L-001〜005）はカバレッジアイテムの遷移を1件ずつ確認する。
// ════════════════════════════════════════════════════════════════
test.describe('ログイン画面 (A101) - 状態遷移テスト', () => {

  // 各テストの開始前にログイン画面を開く。
  // beforeEach を使うことで、テストごとに画面をリセットした状態から始められる。
  test.beforeEach(async ({ page }) => {
    await gotoLoginPage(page);
  });

  // ────────────────────────────────────────────────────────────────
  // TC-L-001: 正常ログイン
  // カバレッジアイテム T1：未ログイン → [入力情報が正しい] → ログイン済み
  //
  // 確認すること：
  //   - TOP画面（/ticket/search）へ遷移すること
  //   - ヘッダに「ようこそ」が表示されること（ログイン状態の証拠）
  // ────────────────────────────────────────────────────────────────
  test('TC-L-001: 正常ログイン', async ({ page }) => {
    await submitLoginForm(page, '0000000001', 'aaaaa11111');

    // 正常ログイン後は TOP画面（チケット検索画面）へ遷移する
    await expect(page).toHaveURL(/\/ticket\/search/);

    // ヘッダの「ようこそ」表示でログイン状態を確認する
    await expect(page.getByText('ようこそ')).toBeVisible();
  });

  // ────────────────────────────────────────────────────────────────
  // TC-L-002: クライアントエラー（会員番号 未入力）
  // カバレッジアイテム T2：未ログイン → [入力情報に誤りあり] → 未ログイン
  //
  // 確認すること：
  //   - ログイン画面に留まること（サーバーへリクエストは送られない）
  //   - クライアントエラー「入力必須項目です。」が表示されること
  //     （ブラウザのバリデーション（Parsley.js）による表示）
  // ────────────────────────────────────────────────────────────────
  test('TC-L-002: クライアントエラー（会員番号 未入力）', async ({ page }) => {
    await submitLoginForm(page, '', 'aaaaa11111');

    // クライアントエラーのためページ遷移は起きず、ログイン画面の URL のまま
    await expect(page).toHaveURL(/\/auth\/login/);

    // ブラウザのバリデーション（Parsley.js）によるエラーメッセージを確認する
    await expect(page.getByText('入力必須項目です。')).toBeVisible();
  });

  // ────────────────────────────────────────────────────────────────
  // TC-L-003: クライアントエラー（パスワード 未入力）
  // カバレッジアイテム T2：未ログイン → [入力情報に誤りあり] → 未ログイン
  //
  // TC-L-002 と同じカバレッジアイテムだが、未入力の項目が異なる。
  // 確認すること：TC-L-002 と同じ
  // ────────────────────────────────────────────────────────────────
  test('TC-L-003: クライアントエラー（パスワード 未入力）', async ({ page }) => {
    await submitLoginForm(page, '0000000001', '');

    await expect(page).toHaveURL(/\/auth\/login/);
    await expect(page.getByText('入力必須項目です。')).toBeVisible();
  });

  // ────────────────────────────────────────────────────────────────
  // TC-L-004: 認証エラー（会員番号 不存在）
  // カバレッジアイテム T3：未ログイン → [会員番号なし / パスワード誤り] → 未ログイン
  //
  // 確認すること：
  //   - サーバーへリクエストが送られ /auth/dologin の URL になること
  //   - サーバーからのエラーメッセージ（.alert）が表示されること
  //
  // TC-L-002/003（クライアントエラー）との違い：
  //   クライアントエラーはブラウザが弾くためサーバーへ届かない。
  //   認証エラーはサーバーまで届いた上で返ってくるエラーである。
  // ────────────────────────────────────────────────────────────────
  test('TC-L-004: 認証エラー（会員番号 不存在）', async ({ page }) => {
    await submitLoginForm(page, '9999999999', 'aaaaa11111');

    // 認証エラーはサーバー処理後にフォワードされるため URL が /auth/dologin になる
    await expect(page).toHaveURL(/\/auth\/dologin/);

    // サーバーから返るエラーメッセージ領域（.alert）が表示されることを確認する
    await expect(page.locator('.alert')).toBeVisible();
  });

  // ────────────────────────────────────────────────────────────────
  // TC-L-005: 認証エラー（パスワード 不一致）
  // カバレッジアイテム T3：未ログイン → [会員番号なし / パスワード誤り] → 未ログイン
  //
  // TC-L-004 と同じカバレッジアイテムだが、失敗の原因が異なる。
  // 「会員番号が存在しない場合」と「パスワードが違う場合」で
  // エラーメッセージが同一であることはセキュリティ上の仕様（どちらで失敗したか
  // 攻撃者に知らせないため）。ここでは表示されることだけを確認する。
  // ────────────────────────────────────────────────────────────────
  test('TC-L-005: 認証エラー（パスワード 不一致）', async ({ page }) => {
    await submitLoginForm(page, '0000000001', 'wrongpasswd');

    await expect(page).toHaveURL(/\/auth\/dologin/);
    await expect(page.locator('.alert')).toBeVisible();
  });
});
```
