# 課題EX-1：Linuxコマンドの基礎を習得する

> **番外編について：** この課題はメインの演習シーケンス（課題1〜8）とは独立しています。Linux 操作に不安がある受講者向けの補足演習です。

**所要時間：** 45〜60 分  
**対象：** Linux コマンドの操作に不安がある受講者  
**前提：**
- WSL2 と Ubuntu のセットアップ（環境構築マニュアル STEP 6）が完了していること
- [WSL2 で Linux コマンドを体験する.md](<../学習資料/WSL2 で Linux コマンドを体験する.md>) を完了していること

---

## この課題の目的

現場でよく使う次の 3 つのスキルを習得します。

1. **ファイルのコピー・移動・削除**
2. **アクセス権（パーミッション）の読み方**
3. **テキスト処理とログ分析**

ゴールは「Linux サーバー上でログファイルを調査し、エラーの件数や内容を特定できること」です。

---

## 準備：Ubuntu のシェルに入る

VS Code の「ターミナル」→「新しいターミナル」を開き、以下のコマンドを実行して Ubuntu のシェルに切り替えます。

```
wsl
```

プロンプトが `username@hostname:/mnt/...` のような形式に変わったら、続けてホームディレクトリに移動します。

```
cd ~
```

プロンプトが `username@hostname:~$` のような形式（`username` は Linux のユーザー名、`hostname` はコンピュータ名）に変わったら準備完了です。

---

## Step 1：ファイルのコピー・移動・削除

### 手順1：前回作成したファイルを確認する

```
ls ~/practice
```

**確認チェックリスト：**

- [ ] `memo.txt` が表示された

> **`memo.txt` が見つからない場合：** 以下のコマンドで再作成してください。
> ```
> mkdir -p ~/practice
> echo "This is a practice memo." > ~/practice/memo.txt
> ```

---

### 手順2：ファイルをコピーする

```
cp ~/practice/memo.txt ~/practice/memo_backup.txt
ls ~/practice
```

**確認チェックリスト：**

- [ ] `memo.txt` と `memo_backup.txt` の 2 つが表示された

> **メモ：** `cp` は「copy」の略です。`cp コピー元 コピー先` の形式で使います。`mv`（move）と違い、元のファイルはそのまま残ります。

---

### 手順3：ディレクトリをまとめてコピーする

```
cp -r ~/practice ~/practice_backup
ls ~
```

**確認チェックリスト：**

- [ ] `practice` と `practice_backup` の両方がホームディレクトリに表示された

> **メモ：** ディレクトリをコピーするときは `-r`（recursive：再帰的）オプションが必要です。ディレクトリは中身を伴う階層構造のため、`-r` を指定することで中身ごと再帰的にコピーします。`-r` なしでディレクトリをコピーしようとするとエラーになります。

---

### 手順4：ファイルを移動・名前変更する

```
mv ~/practice/memo.txt ~/practice/memo_v1.txt
ls ~/practice
```

**確認チェックリスト：**

- [ ] `memo.txt` が消え、`memo_v1.txt` が表示された

> **メモ：** `mv` は同じディレクトリ内で使うとファイル名の変更になります。Windows エクスプローラーで「名前の変更」をした操作と同じです。

---

### 手順5：ファイルを削除する

```
rm ~/practice/memo_backup.txt
ls ~/practice
```

**確認チェックリスト：**

- [ ] `memo_backup.txt` が一覧から消えた

> **注意：** `rm` で削除したファイルはごみ箱に入りません。元に戻せないため、削除前に対象ファイルを必ず確認してください。

---

### 手順6：ディレクトリを削除する

```
rm -r ~/practice_backup
ls ~
```

**確認チェックリスト：**

- [ ] `practice_backup` が一覧から消えた

> **メモ：** `rm -r` の `-r`（recursive：再帰的）オプションはディレクトリを中身ごと削除します。`cp -r` と同様に、ディレクトリを対象にする場合は `-r` が必要です。ファイル単体の削除に `-r` は不要です。なお、`rm` および `rm -r` で削除したファイルやディレクトリはごみ箱に入りません。

---

## Step 2：アクセス権（パーミッション）を読む

Linux ではファイルやディレクトリごとに「誰が」「何をできるか」が設定されています。現場では「スクリプトが実行できない」「ログファイルが読めない」といったトラブルの原因になることがあります。

### 手順7：ls -l の出力を読む

```
ls -l ~/practice
```

出力例：

```
-rw-r--r-- 1 username username 69 May 10 10:00 memo_v1.txt
```

> **メモ：** ファイルサイズ（`69` の部分）は環境によって多少異なる場合があります。`-rw-r--r--` のパーミッション表記が確認できれば問題ありません。

> **パーミッション表記の読み方：**（読みやすくするためにスペースを入れています）
> ```
> - rw- r-- r--
> │ │   │   └── その他のユーザー（読み取りのみ）
> │ │   └────── グループ（読み取りのみ）
> │ └────────── 所有者（読み書き可能）
> └──────────── ファイル種別（- : ファイル、d : ディレクトリ）
> ```
> `r` = 読み取り（read）、`w` = 書き込み（write）、`x` = 実行（execute）

**確認チェックリスト：**

- [ ] ファイル名の左側に `-rw-r--r--` のような記号が表示された

---

### 手順8：実行権限を付与する

シェルスクリプト（自動化コマンドをまとめたファイル）は、実行権限がないと動きません。

```
touch ~/practice/check.sh
ls -l ~/practice/check.sh
chmod +x ~/practice/check.sh
ls -l ~/practice/check.sh
```

**確認チェックリスト：**

- [ ] `chmod` 実行前：`-rw-r--r--` のような表記だった
- [ ] `chmod` 実行後：`-rwxr-xr-x` のように `x` が追加された

> **メモ：** `chmod +x` は「実行権限を追加する」コマンドです。テスト自動化スクリプトをサーバーで動かすとき、実行権限がなくて動かない問題がよく起きます。

## Step 3：テキスト処理とログ分析

現場でよく行う「大量のログから問題のある行を探す」作業を体験します。

### 手順9：サンプルログを作成する

以下をすべて選択してコピーし、ターミナルに貼り付けてください。最後の `EOF` 行まで含めてコピーする必要があります。VS Code のターミナルでは複数行の貼り付け時に確認ダイアログが表示されることがあります。その場合は「貼り付け」を選択してください。

```
cat > ~/practice/app.log << 'EOF'
2024-05-10 10:01:23 INFO  Application started
2024-05-10 10:02:11 INFO  User login: user001
2024-05-10 10:03:45 ERROR Database connection failed: timeout
2024-05-10 10:04:02 INFO  Retry attempt 1
2024-05-10 10:04:15 ERROR Database connection failed: timeout
2024-05-10 10:04:28 INFO  Retry attempt 2
2024-05-10 10:04:41 INFO  Database connection restored
2024-05-10 10:05:00 INFO  User login: user002
2024-05-10 10:06:33 WARN  Disk usage 80%: /var/log
2024-05-10 10:07:45 INFO  Scheduled backup started
2024-05-10 10:08:12 ERROR Permission denied: /backup/data
2024-05-10 10:09:00 INFO  Backup skipped
2024-05-10 10:10:30 INFO  Application health check: OK
EOF
```

**確認チェックリスト：**

- [ ] エラーが表示されずに終了した

---

### 手順10：ファイルの先頭・末尾を確認する

```
head -5 ~/practice/app.log
```

```
tail -5 ~/practice/app.log
```

**確認チェックリスト：**

- [ ] `head` で最初の 5 行が表示された
- [ ] `tail` で最後の 5 行が表示された

> **気づきポイント：** ログファイルは大量の行があることがほとんどです。`head` や `tail` で起動直後のログや最新の状況を素早く確認するのは、現場での基本操作です。

---

### 手順11：特定のキーワードを含む行を抽出する

```
grep "ERROR" ~/practice/app.log
```

**確認チェックリスト：**

- [ ] `ERROR` を含む 3 行だけが表示された

---

### 手順12：行番号付きで検索する

```
grep -n "ERROR" ~/practice/app.log
```

**確認チェックリスト：**

- [ ] 各行の前に行番号（例：`3:...`）が表示された

> **メモ：** `-n`（line number）オプションで行番号を表示できます。問題のある行番号がわかると、原因を素早く特定できます。

---

### 手順13：パイプでコマンドを組み合わせる

パイプ（`|`）は、あるコマンドの出力を次のコマンドに渡す仕組みです。

```
grep "ERROR" ~/practice/app.log | wc -l
```

**確認チェックリスト：**

- [ ] `3`（ERROR の件数）が表示された

> **気づきポイント：** `wc -l` は行数を数えるコマンドです。「本番環境で今日何件のエラーが発生したか」を 1 行で調べられます。

---

### 手順14：指定キーワードを除外して検索する

```
grep -v "INFO" ~/practice/app.log
```

**確認チェックリスト：**

- [ ] `INFO` を含まない行（ERROR、WARN）だけが表示された

> **メモ：** `-v`（invert match）オプションで「一致しない行をすべて表示」します。

---

### 手順15：大文字・小文字を区別せずに検索する

```
grep -i "error" ~/practice/app.log
```

**確認チェックリスト：**

- [ ] `error`（小文字）で検索しても `ERROR`（大文字）の行が表示された

> **メモ：** `-i`（ignore case）オプションで大文字小文字を区別せずに検索できます。ログのフォーマットが統一されていないシステムでも確実に検索できます。

## 演習を終えるとき

演習が終わったら、以下のコマンドを実行して WSL シェルを終了してからターミナルを閉じてください。

```
exit
```

> **注意：** ウィンドウの閉じるボタンを直接押さずに、必ず `exit` で終了する習慣をつけてください。SSH でサーバーに接続して作業する場合も同様で、`exit` による正常終了はサーバー管理の基本作法です。
