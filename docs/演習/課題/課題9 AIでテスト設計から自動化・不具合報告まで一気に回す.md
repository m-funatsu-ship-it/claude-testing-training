# 課題9：AIでテスト設計から自動化・不具合報告まで一気に回す

**所要時間：** 50〜65 分  
**対象：** 課題1〜8を終えた受講者  
**前提：**
- 正常版 ATRS（`http://localhost:8082/atrs/`）とバグ版 ATRS（`http://localhost:8083/atrs/`）が起動していること
- [ATRS環境の使い分け.md](../../ATRS/環境構築/ATRS環境の使い分け.md) を確認済みであること

> **バグ版を初めて起動する場合**  
> [ATRS ローカル起動手順.md](../../ATRS/環境構築/ATRS%20ローカル起動手順.md) の「4. データベースの準備」を参照して `mvn sql:execute -f atrs-initdb/pom.xml` を先に実行してください。  
> その後、以下の順で起動してください。
> ```
> mvn clean install
> mvn cargo:run -f atrs-web/pom.xml
> ```

> **ターミナルを2つ開いておく**  
> 正常版とバグ版を同時に起動したまま保持するため、VS Code のターミナルを2つ用意してください。  
> ターミナルパネルの `+` ボタンで追加できます。

---

## この課題の目的

課題1〜8では、テスト設計技法・APIテスト・テストモデル・Playwright自動化を個別に体験しました。

この課題では、それらを**ひとつの流れとして実践**します。

> 仕様を受け取る → AIで設計する → AIで自動化する → 不具合を発見して報告する

これは現場の企業が実際に取り組んでいる「AI主導のテストプロセス」の体験版です。  
ゴールは「AIに指示を出しながら、テストプロセス全体を自分で回せた」という感覚をつかむことです。

---

## この課題で発見を狙う不具合

バグ版（Port: 8083）には、チケット予約の**合計金額計算に関する重大な不具合**が仕込まれています。

この不具合は画面を目で見るだけでは気づきにくく、**自動テストが特に効果を発揮するケース**です。

> **課題1との接続：** 課題1で学んだ「運賃算出ロジック」が今回の題材です。  
> 設計技法の知識が自動テストに直結する体験をしてください。

## Step 1：Claude Code でシーケンス図とテスト条件を同時に生成する（12分）

Claude Code を起動し、以下のプロンプトをコピーして送ってください。

```
コードは書かずに、設計のみ行ってください。

以下はATRS（航空チケット予約システム）のチケット予約機能の仕様です。
この仕様から以下の2つを同時に作成してください。

【作成1】Mermaid の sequenceDiagram 形式でシーケンス図（テストモデル）を作成してください。
- ログイン済みユーザーが片道フライトを予約する流れを表現してください
- 画面遷移（B101→B202→B203→B204/B205）とシステムの処理を含めてください

【作成2】合計金額算出ルールに関するテスト条件の一覧を作成してください。
- 有効同値パーティション（正常系）と無効同値パーティション（異常系）に分類してください
- 同値分割法の観点で、年齢による分類（12歳以上・12歳未満）を含めてください

【仕様：合計金額の算出方法（片道）】
合計金額 = 運賃 × 搭乗者数（12歳以上）
         + （基本運賃 × 60% − 運賃種別ごとの割引額）× 搭乗者数（12歳未満）

端数処理：合計金額の100円未満は切り上げとする

【仕様：予約フローの画面遷移】
- B101（空席照会画面）でフライトを選択し「予約」ボタンを押す
- ログイン済みの場合、B202（お客様情報入力画面）へ直接遷移する
- B202で搭乗者情報と予約代表者情報を入力し「予約確認」ボタンを押す
- B203（申し込み内容確認画面）で合計金額と搭乗者情報を確認し「予約確定」ボタンを押す
- 予約成功時はB204（予約完了画面）へ遷移し、予約番号・合計金額・支払期限が表示される
- 予約失敗時はB205（予約失敗画面）へ遷移する
```

出力されたら、以下の点だけ確認してください（2分）。

| 確認項目 | 確認できたか |
|----------|------------|
| シーケンス図にB204（予約完了）とB205（予約失敗）の両方が含まれているか | |
| テスト条件に「12歳未満」と「12歳以上」の分類があるか | |
| テスト条件に「端数切り上げ」の観点があるか | |

不足があれば追加プロンプトを送って補完してもらいましょう。

## Step 2：Claude Code でテストケースを設計・レビューする（10分）

続けて以下のプロンプトを送ってください。

```
Step 1で作成したテスト条件をもとに、合計金額の算出に関するテストケースを作成してください。

作成後、ISO/IEC/IEEE 29119-4（テスト設計技法の国際規格）およびJSTQB（ソフトウェアテスト技術者資格）の観点でセルフレビューを行い、以下の技法が適切に適用されているか確認してください。不足があればテストケースを追加してください。
1. 同値分割法（12歳以上・12歳未満の分類）
2. 境界値分析（年齢の境界：11歳・12歳・13歳）

テストケースは表形式で整理してください。
列：テストケースID・搭乗者の年齢構成・期待する合計金額の考え方
```

出力されたテストケース表を確認してください。

確認できたら、以下のプロンプトを送って設計成果物をファイルに保存してください。

```
Step 1とStep 2で作成した以下の内容を docs/演習/結果/課題9 テスト設計.md としてまとめて保存してください。
- シーケンス図（Mermaid コードのまま）
- テスト条件一覧
- テストケース表
```

> **次のStepで使うもの：**  
> 「大人1名（20歳）＋子供1名（5歳）」の組み合わせのテストケースを1件、頭に入れておいてください。  
> 全テストケースを手動実行する必要はありません。Playwright が自動で実行します。

## Step 3：Claude Code で Playwright テストコードを生成する（10分）

> **セッション使用量について**  
> Claude Code はテストコードを生成する際に、アプリのソースコードを複数ファイル読み込む場合があります。そのため、このステップでセッションの使用量が増加することがあります。  
> 使用上限に達した場合は、新しい会話を開始してから以下を伝えて続けてください。
> ```
> docs/演習/結果/課題9 テスト設計.md を読んでください。
> ```

以下のプロンプトを Claude Code に送ってください。

```
以下の条件で Playwright のテストコードを tests/reserveFare.spec.ts として作成してください。

【テスト対象】
ATRSチケット予約フロー（子供料金の適用確認）
baseURL: http://localhost:8082/atrs/
テスト用会員番号: 0000000001 / パスワード: aaaaa11111

【テストケース: TC-R-001 子供料金の適用確認】
1. ログインして空席照会画面に遷移する
2. 出発地「東京（羽田）」・到着地「大阪（伊丹）」・片道・搭乗日は7日後で検索する
3. 検索結果から最初のフライトを選択し「予約」ボタンを押す
4. お客様情報入力画面（B202）で「搭乗者を追加」ボタンを押して搭乗者2を追加する
5. 搭乗者2に名前（カタカナ、任意の名前でよい）・年齢「5」・性別「男性」を入力する
6. 「予約確認」ボタンを押す
7. 申し込み内容確認画面（B203）で以下を確認する
   - フライト情報一覧に表示されている運賃（大人1名分）の金額を取得する
   - 合計金額を取得する
   - 合計金額が「大人運賃 × 2」より少ないことを確認する
     （子供料金は大人より安いため、大人2名分より合計が小さくなるはずである）

【補足】
- 金額テキストは「¥12,345」形式のため、数値に変換して比較してください
- playwright.config.ts の baseURL は http://localhost:8082/atrs/ に設定してください
```

> **playwright.config.ts がすでにある場合**  
> 上書きを確認してきたら「上書きしてください」と伝えてください。

コードが生成されたら次のStepへ進んでください。中身は読まなくて構いません。

## Step 4：正常版で実行してからバグ版で実行する（15分）

### 正常版で実行する（期待動作の確認）

VS Code のターミナルを開き（`Ctrl+Shift+@`）、以下を実行してください。

```
npm run test:headed -- tests/reserveFare.spec.ts
```

ブラウザが自動操作されて予約フローが動く様子を確認してください。  
`1 passed` が表示されれば、正常版では期待通りに動作しています。

> **テストが失敗した場合**  
> Claude Code に「エラーメッセージを確認して修正してください」と伝えてください。  
> セレクタや画面の操作手順を Claude Code が自律的に修正します。

### バグ版で実行する（不具合の検出）

接続先をバグ版（Port: 8083）に切り替えて実行します。  
以下のプロンプトを Claude Code に送ってください。

```
tests/reserveFare.spec.ts の baseURL を http://localhost:8083/atrs/ に変更して、
テストを実行してください。テストが終わったら結果をそのまま表示してください。
```

テストが失敗（FAIL）したことを確認してください。  
Claude Code の返答に表示された以下の情報を次のStepで使います。

- `Expected`（期待値）
- `Received`（実際の値）

続けて、以下のプロンプトを Claude Code に送って baseURL を正常版に戻してください。

```
tests/reserveFare.spec.ts の baseURL の上書き設定を削除して、
playwright.config.ts の設定（http://localhost:8082/atrs/）を使う状態に戻してください。
```

## Step 5：Claude Code で不具合レポートを作成して Git にコミットする（10分）

### 不具合レポートの作成

以下のプロンプトをコピーし、`（ここに Step 4 で Claude Code が返答した FAIL のテキストを貼り付けてください）` の部分を実際のテスト失敗テキストに置き換えてから、Claude Code に送ってください。

```
以下の Playwright テスト失敗結果をもとに、不具合レポートのドラフトを作成し、
docs/演習/結果/課題9 不具合レポート.md として保存してください。

【不具合レポートに含める項目】
- 不具合ID・タイトル
- 重大度（severity）：A〜Dで判定してください（A=金銭・個人情報への直接的影響、B=主要機能の全停止、C=一部機能の不全、D=軽微な表示問題）
- 再現条件
- 再現手順（Playwright が実行した手順をもとに）
- 発生事象（Actual）
- 期待結果（Expected）
- 使用環境（バグ版 Port: 8083）

【テスト失敗結果】
（ここに Step 4 で Claude Code が返答した FAIL のテキストを貼り付けてください）
```

> **貼り付け例：**  
> ```
> ✗ TC-R-001: 子供料金の適用確認 (16.9s)
> Error: expect(received).toBeLessThan(expected)
> Expected: < 41200
> Received:   41200
> ```

出力された不具合レポートを確認し、内容が正しければ次へ進んでください。
