# WSL2 で Linux コマンドを体験する

**所要時間：** 30〜40 分  
**対象：** STEP 6（WSL2（ローカル Ubuntu）のセットアップ）を完了した受講者  
**前提：**
- WSL2 と Ubuntu のセットアップ（環境構築マニュアル STEP 6）が完了していること

---

## この演習の目的

Linux はサーバー環境で最も広く使われている OS です。テストエンジニアとして現場に入ると、ログ確認・ファイル操作・サービス管理などを Linux コマンドで行う場面があります。  
この演習のゴールは「マウスを使わずに Ubuntu 上で基本的なファイル操作・パッケージ管理ができること」です。  
コマンドを暗記する必要はありません。まず「動かしてみること」に集中してください。

---

## ① ファイルシステムの探索

Linux のファイルシステムは Windows（`C:\Users\...`）とは構造が異なります。  
まず自分の「現在地」を把握することから始めます。

### 手順1：Ubuntu のターミナルを開き、ホームディレクトリに移動する

VS Code の「ターミナル」→「新しいターミナル」を開き、以下のコマンドを実行して Ubuntu のシェルに切り替えます。

```
wsl
```

プロンプトが `username@hostname:/mnt/c/...` のような表示に変わったら Ubuntu のシェルに切り替わっています。

> **メモ：** `wsl` を実行すると、VS Code で開いているフォルダが `/mnt/` 以下にマウントされたパスからスタートします。演習はホームディレクトリで行うため、次のコマンドで移動します。

続けて以下のコマンドを実行してください。

```
cd ~
pwd
```

**確認チェックリスト：**

- [ ] `/home/username` のようなパスが表示された

> **メモ：** `~`（チルダ）は Linux でホームディレクトリを表す記号です。`cd ~` で常にホームディレクトリに戻れます。`pwd`（Print Working Directory）は「今いる場所」を表示するコマンドです。Linux のパスはスラッシュ（`/`）区切りで、ドライブ文字（`C:\`）はありません。

---

### 手順2：ディレクトリの内容を確認する

```
ls -la
```

**確認チェックリスト：**

- [ ] ドット（`.`）から始まるファイルやディレクトリが表示された

> **メモ：** `-la` は「詳細情報（`-l`）と隠しファイルを含む全ファイル（`-a`）を表示する」オプションです。`-a` なしの `ls` では通常ファイルしか表示されません。Windows エクスプローラーの「隠しファイルを表示する」に相当します。ドットから始まるファイルが Linux の「隠しファイル」です。

---

### 手順3：練習用ディレクトリを作成して移動する

```
mkdir practice
cd practice
pwd
```

**確認チェックリスト：**

- [ ] `pwd` で `/home/username/practice` が表示された

> **`practice` ディレクトリが既に存在する旨のエラーが表示された場合：** エラーは無視して、`cd practice` と `pwd` をそのまま実行してください。`practice` ディレクトリへの移動と現在地の確認に問題はありません。

> **メモ：** `mkdir` はディレクトリ（フォルダ）の作成、`cd` は移動です。Windows の「新しいフォルダーを作成 → フォルダーを開く」に相当します。

---

## ② ファイルの作成と編集

### 手順4：空のファイルを作成する

```
touch memo.txt
ls
```

**確認チェックリスト：**

- [ ] `memo.txt` が一覧に表示された

> **メモ：** `touch` はファイルを新規作成するコマンドです（既にあるファイルに使うとタイムスタンプが更新されます）。中身が空のファイルを素早く作りたいときに使います。

---

### 手順5：ファイルを編集する

`nano` はターミナル上で動作するテキストエディタです。

```
nano memo.txt
```

エディタが開いたら、以下のように入力してください。

```
This is a practice memo for Linux commands.
Environment: WSL2 Ubuntu
```

入力後、`Ctrl + X` → `Y` → `Enter` の順に押して保存・終了します。

**確認チェックリスト：**

- [ ] ターミナルに戻った（エディタが閉じた）

---

### 手順6：ファイルの内容を確認する

```
cat memo.txt
```

**確認チェックリスト：**

- [ ] 手順5で入力した 2 行が英語で表示された

---

### 手順7：ファイルの内容を検索する

```
grep "practice" memo.txt
```

**確認チェックリスト：**

- [ ] `practice` を含む行だけが表示された

> **気づきポイント：** `grep` はログファイルから特定のエラーメッセージを探すときに頻繁に使うコマンドです。「大量のログから特定のキーワードを含む行だけ抽出する」場面でテスト作業でも活躍します。

## ③ パッケージ管理とサービスの状態確認

Linux でソフトウェアをインストールするには `apt`（パッケージマネージャー）を使います。Windows でいう「インストーラーをダウンロードして実行する」作業を、コマンド 1 行で行えます。

### 手順8：パッケージリストを更新する

```
sudo apt update
```

Linux ユーザーのパスワードを求められたら、STEP 6 のセットアップ時に設定したパスワードを入力してください（入力中は画面に表示されません）。

**確認チェックリスト：**

- [ ] エラーなく完了した

> **メモ：** `sudo` は管理者権限でコマンドを実行する指示です。Windows の「管理者として実行」に相当します。パッケージリストの更新やソフトウェアのインストールには管理者権限が必要です。

---

### 手順9：nginx をインストールする

nginx（エンジンエックス）は世界中で広く使われている Web サーバーです。ここではパッケージのインストールからサービスの起動確認まで、一連の流れを体験します。

```
sudo apt install -y nginx
```

**確認チェックリスト：**

- [ ] エラーなくインストールが完了した

> **メモ：** 初回インストール時は `Setting up nginx` のようなメッセージが表示されます。既にインストール済みの場合は `nginx is already the newest version` のように表示されます。どちらもエラーなく完了していれば問題ありません。

---

### 手順10：nginx の状態を確認する

```
sudo systemctl status nginx
```

**確認チェックリスト：**

- [ ] `Active: active (running)` と表示された

> **メモ：** `systemctl` は Linux 上で動いているサービスを管理するコマンドです。テスト現場では「デプロイ後にサービスが正常起動しているか確認する」用途でよく使います。出力が長い場合はページャー（画面送り表示）が起動します。内容を確認したら `q` を押して終了してください。

> **トラブルシューティング**
> `System has not been booted with systemd` というエラーが表示された場合は、`sudo service nginx status` を代わりに使用してください。この場合は `* nginx is running` のような表示が確認できれば正常です。

## 演習を終えるとき

演習が終わったら、以下のコマンドを実行して WSL シェルを終了してからターミナルを閉じてください。

```
exit
```

> **注意：** ウィンドウの閉じるボタンを直接押さずに、必ず `exit` で終了する習慣をつけてください。SSH でサーバーに接続して作業する場合も同様で、`exit` による正常終了はサーバー管理の基本作法です。
