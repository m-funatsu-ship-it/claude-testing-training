# 課題9 不具合レポート

## BUG-001：子供搭乗者に大人運賃が適用される（子供割引が機能しない）

| 項目 | 内容 |
|------|------|
| 不具合ID | BUG-001 |
| タイトル | 子供搭乗者に大人運賃が適用され、合計金額が誤って算出される |
| 重大度 | **A（金銭への直接的影響）** |
| 報告日 | 2026-05-20 |
| 検出手段 | Playwright 自動テスト（TC-R-001） |

---

## 重大度の判定理由

子供料金の割引が適用されず、本来より高い金額が請求される。
ユーザーは誤った金額で予約を確定させられる可能性があり、**金銭的な過剰請求**に直結するため重大度 A と判定する。

---

## 再現条件

| 条件 | 内容 |
|------|------|
| 対象環境 | バグ版 ATRS（http://localhost:8083/atrs/） |
| ブラウザ | Chromium |
| ログインアカウント | 会員番号: 0000000001 / パスワード: aaaaa11111 |
| フライト条件 | 東京(羽田) → 大阪(伊丹)・片道・一般席 |
| 搭乗者構成 | 大人1名（会員本人）＋子供1名（年齢: 5歳） |

---

## 再現手順

1. `http://localhost:8083/atrs/auth/login?form` にアクセスし、会員番号 `0000000001` / パスワード `aaaaa11111` でログインする
2. 空席照会画面（B101）で以下の条件を入力し「照会」ボタンを押す
   - フライト種別: 片道
   - 区間: 東京(羽田) → 大阪(伊丹)
   - 搭乗日: 当日から7日後
   - 搭乗クラス: 一般席
3. 検索結果の一覧から最初のフライトを選択し「選択フライトを予約」ボタンを押す
4. お客様情報入力画面（B202）で「搭乗者を追加」ボタンを押す
5. 搭乗者2に以下の情報を入力する
   - お名前（カタカナ）: テスト タロウ
   - 年齢: **5**
   - 性別: 男性
6. 「予約確認」ボタンを押す
7. 申し込み内容確認画面（B203）に表示される合計金額を確認する

---

## 発生事象（Actual）

- 大人1名分の運賃: **¥20,600**
- 合計金額（大人1名＋子供1名）: **¥41,200**
- 合計金額が「大人運賃 × 2」と**等しく**、子供割引が一切適用されていない

```
Error: expect(received).toBeLessThan(expected)
Expected: < 41200
Received:   41200
```

---

## 期待結果（Expected）

仕様「合計金額 = 運賃 × 搭乗者数（12歳以上）＋（基本運賃 × 60% − 割引額）× 搭乗者数（12歳未満）」に基づき、以下の金額になるはずである。

- 大人1名分: ¥20,600
- 子供1名分（12歳未満）: 基本運賃 × 60% − 割引額（¥20,600 より小さい値）
- 合計金額: **¥41,200 未満**
